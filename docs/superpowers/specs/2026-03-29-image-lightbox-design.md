# Image Lightbox — Twitter Split Design

**Date:** 2026-03-29
**Status:** Approved

## Overview

Переработка компонента `ImageLightbox` — замена простого чёрного оверлея с картинкой по центру на полноэкранный интерфейс в стиле Twitter/X: изображение слева, боковая панель с постом и комментариями справа.

## Layout

Два флекс-блока на всю высоту viewport:

- **Левая часть (image area):** `flex: 1` — занимает весь остаток пространства. Чёрный фон, изображение центрировано с `object-fit: contain`.
- **Правая панель (sidebar):** `width: 36%`, `min-width: 300px`, `max-width: 420px`. Тёмный фон `#000`, граница слева `1px solid #2f3336`.

## Навигация между изображениями

Когда пост содержит несколько изображений:
- Стрелки `‹` `›` расположены по бокам image area, вертикально по центру. Фон `rgba(255,255,255,0.12)`, border-radius 50%.
- Точки-индикаторы снизу по центру image area — активная точка белая, остальные полупрозрачные.
- При одном изображении стрелки и точки скрыты.

## Боковая панель — содержимое

Сверху вниз:
1. Аватар + имя + хэндл автора поста
2. Полный текст поста
3. Дата и время публикации
4. Статистика: репосты, лайки, просмотры
5. Кнопки действий: ответить, репост, лайк, поделиться
6. Заголовок "Комментарии"
7. Список комментариев (infinite scroll)
8. Фиксированная строка ответа (`position: sticky bottom`)

## Комментарии — Infinite Scroll

- Панель прокручивается независимо от image area.
- При достижении нижней границы — показывается спиннер, выполняется запрос следующей страницы.
- Комментарии — отдельный API-запрос с пагинацией (cursor-based), независимый от данных поста.

## Данные

Lightbox получает `postId` (не только `src`). Данные поста берутся из уже загруженного состояния (пропс или контекст) — дополнительного запроса для поста не нужно. Комментарии подгружаются лениво при открытии.

## Закрытие

- Кнопка `✕` — абсолютно позиционирована, вверху слева над image area.
- Клик на тёмную область вне изображения (сама image area, не sidebar).
- Клавиша `Escape`.
- Кнопка "Назад" браузера (`history.back()` или `pushState` при открытии).

## Интерфейс компонента

```tsx
interface ImageLightboxProps {
  src: string;           // URL текущего изображения
  allSrcs?: string[];    // Все изображения поста (для навигации)
  postId: string;        // ID поста для загрузки данных и комментариев
  post: Post;            // Данные поста (уже загружены в родителе)
  onClose: () => void;
}
```

## Out of Scope

- Зум/масштабирование изображения
- Скачивание изображения
- Видео в lightbox
- Мобильный swipe-жест (отдельная задача)
